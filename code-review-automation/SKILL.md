## What I do

I automate code review by:

- Analyzing PRs against defined coding standards
- Identifying potential breaking changes
- Detecting security vulnerabilities
- Checking test coverage and documentation
- Suggesting improvements for code quality
- Assigning reviewers based on changed files
- Generating comprehensive review summaries

## When to use me

Use this skill when:
- Setting up automated code review in CI/CD
- Need consistent review standards across PRs
- Want to reduce reviewer burden on routine checks
- Preparing PRs for manual review
- Tracking code review metrics

## Usage

```bash
skill code-review-automation --pr 123
skill code-review-automation --analyze --output review.json
skill code-review-automation --comment --pr 123
```

## Review Checklist

I verify:
- Code style compliance
- Test coverage requirements
- Documentation updates
- Security vulnerability scan
- Breaking change detection
- Performance impact assessment
- Architecture pattern adherence
